# ありがとうカード プロジェクト — ドキュメントカタログ

このディレクトリには、ありがとうカードプロジェクトのドキュメントをAIが消費しやすい形式でまとめたMarkdownファイル群が含まれています。

## プロジェクト概要（1行）

FREEMIND社の全社員60名が使う社内感謝カードWebアプリ。感謝メッセージに対しAIが自動でコアコンピテンシー（5種）をラベル付けする。若手13名のチームがアジャイルで約6ヶ月かけてMVPを開発する。

## ドキュメント一覧

### project-overview.md
**概要:** プロジェクトの背景・目的・ビジョン・コアアイデア・進め方の全体像
**推奨シーン:** プロジェクトの全体像を把握したいとき。新メンバーへの説明。プロダクトの目的や価値を確認したいとき。
**元ファイル:** docs/kickoff.html, index.html

### competencies.md
**概要:** 5つのコアコンピテンシー（誠実・当事者意識・挑戦への意欲・協働・創造的思考）の詳細定義とDo/Don't行動例
**推奨シーン:** AIラベリングのプロンプト設計時。コンピテンシーの判定基準を確認したいとき。感謝メッセージの分類ロジックを検討するとき。
**元ファイル:** references/ideal-candidate-profile.md

### team.md
**概要:** 13名のチーム構成、各フェーズの役割分担、横断的な役割（PO/TL/SM等）
**推奨シーン:** チーム体制や担当者を確認したいとき。誰に何を聞けばよいか判断するとき。プロジェクト管理の文脈で。
**元ファイル:** docs/team.html, docs/inception-deck.html

### development-guide.md
**概要:** アジャイル/スクラムによる開発プロセス全体。Sprint 0の内容と進め方、Sprint 1以降のスプリントサイクル、役割定義、WF→アジャイル対応表
**推奨シーン:** 開発プロセスや進め方を確認したいとき。スプリント計画を立てるとき。スクラムの各イベントの意味を確認したいとき。
**元ファイル:** docs/dev-guide.html

### inception-deck.md
**概要:** インセプションデッキ10枚の内容。エレベーターピッチ、パッケージデザイン（決定済み）、ステークホルダーマップ、トレードオフスライダー、リソース計画。一部TBD項目あり
**推奨シーン:** プロジェクトの方向性・スコープ・制約を確認したいとき。トレードオフの判断基準を知りたいとき。ステークホルダーを把握したいとき。
**元ファイル:** docs/inception-deck.html

### sprint0.md
**概要:** Sprint 0前半で実施するインセプションデッキとデザインスプリントの一般的な進め方の解説。10のタフクエッション、デザインスプリント5日間の流れ
**推奨シーン:** インセプションデッキやデザインスプリントのやり方を学びたいとき。Sprint 0の進め方を詳しく知りたいとき。
**元ファイル:** references/sprint0-first-half.html

## 制約・前提条件のサマリ

| 項目 | 値 |
|------|-----|
| 対象ユーザー | FREEMIND社 全社員60名（4部署: 営業4名、開発52名、DX企画2名、管理3名） |
| チーム規模 | 13名（PO/TL 1名、SM 1名、企画リード 1名、開発メンバー 10名） |
| 開発手法 | アジャイル/スクラム、2週間スプリント |
| 開発期間 | 約6ヶ月（目安、硬い締切ではない） |
| インフラ予算 | 月額 50 USD（インフラ + AI API） |
| インフラ | AWS |
| 認証 | Google OAuth（@freemind.co.jp） |
| 対応端末 | レスポンシブ（PC・スマートフォン両対応必須） |
| コアコンピテンシー | 誠実、当事者意識、協働、挑戦への意欲、創造的思考 |
| 最優先事項 | メンバーの学び・成長（納期やスコープより優先） |
