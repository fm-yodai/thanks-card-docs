# インセプションデッキ

ありがとうカード プロジェクトの全体像をチームで共有し、合意するための10枚のスライド。Sprint 0 の最初に実施。

## 01. 我々はなぜここにいるのか（WHY ARE WE HERE）

日々の業務で埋もれる「ありがとう」を可視化し、感謝を通してコアコンピテンシーを浸透させる。

- 日々の業務の忙しさの中で感謝の気持ちは表面化されにくい。ストレスや多忙により、忘れてしまう感謝もある。
- 組織力向上フォーラムが Lighthouse や社員の声をもとに策定した5つのコアコンピテンシー（誠実、当事者意識、協働、挑戦への意欲、創造的思考）は周知されているものの、**日常業務で意識されることはほとんどない**。「理想論」として受け止められている側面もある。
- **本プロジェクトでは、社員が自然にコンピテンシーを意識し、体現できる状態を目指す。**

**もう一つの目的:** 若手メンバーが0からプロダクトを作り切る経験を通じて成長すること。このプロジェクトは「プロダクト」と「人」の両方を育てる取り組み。

## 02. エレベーターピッチ（ELEVATOR PITCH）

> **日々の感謝を伝えそびれている** FREEMIND の全社員のための
> **「ありがとうカード」**というプロダクトは、
> **社内向け感謝カード x AI ラベリングアプリ**です。
>
> これは**感謝メッセージを送るだけで、AIが自動でコアコンピテンシーをラベル付け**でき、
> **Chatwork でのお礼や対面での「ありがとう」**とは違って、
> **感謝が記録として蓄積され、コンピテンシーが自然と共通言語になる**ことが特徴です。

### なぜ外部SaaSではなく自社開発なのか

- **若手の成長機会** — 若手は0からプロダクトを作り切る経験ができない。企画・設計・開発・リリース・運用の全工程を若手メンバーが体験する。
- **自社文化への最適化** — 5つのコアコンピテンシーに完全に合わせたAIラベリングは、汎用SaaSでは実現できない。

## 03. パッケージデザイン（PRODUCT BOX）

もしこのプロダクトを一つの箱に入れて誰かに手渡すなら、どんな言葉を添えるか？

コンセプト: **忙しさの中で忘れかけた温もりを、そっと思い出させてくれるもの** — そんな存在を目指して。

### 選定コンセプト: 「種まき」（決定済み）

> **今日の「ありがとう」は明日の土壌になる。**
>
> 日々の「助かった」を言葉にする。ただそれだけのことが、気づけば組織の根を静かに強くしていきます。

特徴:
- 手のひらから届く
- 習慣が根を張る
- 空気が変わる

チーム議論の結果、「種まき」コンセプトを採用。日常の感謝が組織の土壌となり、根を張っていくというメタファーがプロダクトのビジョンに最もフィットするため。

## 08. 期間を見極める（SIZE IT UP）

約6ヶ月でMVPリリースを目指す。ただし硬い締切ではなく、学びを優先した目安として設定。

| フェーズ | 期間 | 内容 |
|---------|------|------|
| Sprint 0 | 1〜2ヶ月 | 準備期間：インセプションデッキ、UX設計、技術選定・環境構築、バックログ作成、DoD策定 |
| Sprint 1〜2 | — | コア機能開発：カード送受信 → AIラベリング → タイムライン表示 |
| Sprint 3〜5 | — | フィードバック & 拡充：UI改善、通知機能、AI精度チューニング |
| Sprint 6（目標） | — | ベータ → MVPリリース：先行テスト後、全社60名へ展開 |

- **タイムラインの硬さ: 目安** — 6ヶ月は目安であり、硬い締切ではない。学びが最優先なので、必要に応じてスコープを調整して対応する。
- **スプリントの長さ: 2週間** — 2週間を1サイクルとして毎回「動くもの」を作る。途中で方向転換もOK。大事なのは学びのスピード。

## 09. トレードオフスライダー（TRADE-OFF SLIDERS）

すべてを最高にはできない。チームで優先順位を合意し、判断に迷ったときの羅針盤にする。以下はPOの初期案。

| 項目 | 優先度 |
|------|--------|
| 学び | MAX（最優先） |
| 品質 | 高め |
| 納期 | 中程度 |
| スコープ | 中程度（調整弁） |

- **学び = 最優先** — メンバーの成長機会がプロジェクトの最大目的。納期やスコープは多少犠牲にしても、学びの質を落とさない。
- **スコープ = 調整弁** — 遅延が発生したらスコープを削って対応。機能を減らしてでも、学びと品質を守る。

### 議論ポイント

- Q1. このスライダーの位置に違和感はある？特に品質と納期のバランスについて。
- Q2. 「学びのために納期を延ばす」はどこまで許容できる？稟議との兼ね合いは？
- Q3. 予算（50 USD/月）は固定制約？それとも交渉の余地がある？
