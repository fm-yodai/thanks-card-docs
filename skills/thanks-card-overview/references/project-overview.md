# ありがとうカード プロジェクト概要

## プロジェクトの趣旨

若手メンバー主導で、感謝が循環する社内Webアプリをつくるプロジェクト。

## 背景：いま、組織に足りていないもの

日々の業務の中で「ありがとう」を感じても、伝える機会は意外と少ない。感謝の気持ちが埋もれてしまうことで、以下の課題が見えてきた。

1. **感謝が言語化されない** — 日常の「助かった！」がそのまま流れてしまい、言葉として届かないケースが多い。
2. **部署を超えた繋がりが弱い** — 全社60名、チームをまたいで感謝を届ける仕組みがなく、縦割り意識が生まれやすい。
3. **エンゲージメントが見えにくい** — 組織のコンディションを定量的に把握する手段が限られ、施策が打ちにくい。

## コアアイデア

ありがとうカードを送るとき、メッセージの内容から**生成AIが自動でコンピテンシーをラベル付け**する。

例：「急な依頼にも快く対応してくれてありがとう！」と送ると → 「協働」「誠実」がタグとして付く。

自然に使っているうちに、コンピテンシーが共通言語として根付いていくことを目指す。

## 5つのコアコンピテンシー

会社が期待する人材像を日常に溶け込ませるため、以下の5つを「共通言語」にする。

1. **誠実**
2. **当事者意識**
3. **協働**
4. **挑戦への意欲**
5. **創造的思考**

組織力向上フォーラムが Lighthouse（会社理念）や社員の声をもとに策定したもの。研修やスローガンだけでは浸透しないため、毎日の「ありがとう」に紐づける。

## プロダクトビジョン：つくるもの

全社員が誰にでも「ありがとう」を送れるWebアプリ。AIと分析で、感謝を組織の力に変える。

### コア機能

- **AI自動コンピテンシーラベリング** — 感謝メッセージを入力するだけで、生成AIが最も近いコアコンピテンシーを判定。送る側も受け取る側も、自然とコンピテンシーを意識できるようになる。
- **部署別 月次レポート** — 部署ごとのカード送受信状況やコンピテンシー分布を自動レポート化。エンゲージメント調査の指標としても活用。
- **分析パイプライン** — 蓄積されたデータから組織の傾向を可視化。どのコンピテンシーが活発か、どの部署間の交流が多いかなどを分析。

## なぜ若手メンバーが中心なのか

このプロジェクトは、プロダクトをつくるだけではなく、メンバー自身の成長機会でもある。

1. **企画から運用まで一気通貫** — 要件定義・設計・開発・リリース・運用改善。プロダクト開発の全工程を自分たちの手で経験できる。
2. **生成AI x プロダクトの実践** — LLMを使ったコンピテンシー判定の実装を通じて、生成AIのプロダクト活用スキルが身につく。
3. **データ分析・可視化** — 月次レポートや分析パイプラインの構築を通じて、データエンジニアリングの基礎を実践的に学べる。
4. **全社を巻き込むプロジェクト** — 60名全員が使うアプリを届ける。社内のステークホルダーとの調整やフィードバック対応など、リアルなプロダクトマネジメントを体験できる。

## 進め方の概要

アジャイルで、約半年後のMVPリリースを目指す。小さく作って、早くフィードバックをもらう。

| フェーズ | 期間 | 内容 |
|---------|------|------|
| Phase 1 | Month 1-2 | 企画・設計フェーズ：ユーザーストーリー整理、技術選定、UIデザイン、AIプロンプト設計 |
| Phase 2 | Month 2-4 | 開発スプリント：2週間スプリントで機能を積み上げ。カード送受信 → AIラベリング → タイムライン表示の順で構築 |
| Phase 3 | Month 4-5 | 社内ベータテスト：一部メンバーに先行公開してフィードバック収集。UIの改善やAI精度のチューニング |
| Phase 4 | Month 6 | MVPリリース：全社60名へ展開。月次レポート機能も含めた最小限のプロダクトとして公開 |

アジャイルで進めるため、完璧を目指すのではなく、各スプリントで動くものをつくり、使う人の声を聞きながら育てていく。途中で方向転換してもOK。大事なのは学びのスピード。

## プロジェクトの価値観

- **つくりながら学ぶ**
- **声を聞いて改善する**
- **失敗OK、挑戦しよう**
- **全社60名に届けよう**
