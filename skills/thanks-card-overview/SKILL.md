# ありがとうカード プロジェクトドキュメント

FREEMIND社の社内感謝カードWebアプリ。感謝メッセージに対しAIが5つのコアコンピテンシーを自動ラベル付けする。若手13名がアジャイルで約6ヶ月かけてMVPを開発中。

## ドキュメント一覧

質問内容に応じて必要なファイルを読んでください。

| ファイル | 内容 | こんなときに読む |
|---------|------|----------------|
| [inception-deck.md](references/inception-deck.md) | インセプションデッキ10枚（エレベーターピッチ、トレードオフ等） | プロジェクトの全体像、目的、スコープ・制約・トレードオフの確認 |
| [competencies.md](references/competencies.md) | 5つのコアコンピテンシーの定義とDo/Don't行動例 | AIラベリング設計、判定基準、感謝メッセージの分類 |
| [development-guide.md](references/development-guide.md) | スクラム開発プロセス、スプリントサイクル、役割定義 | 開発の進め方、スプリント計画 |
| [team.md](references/team.md) | 13名のチーム構成、各フェーズの役割分担 | 担当者の確認、誰に何を聞くか |
| [design-system.md](references/design-system.md) | デザインシステム定義（コンセプト・トークン・コンポーネント・パターン・Pencil連携・保守運用） | デザインシステムの仕様、UIコンポーネント、トークン定義、命名規約、Pencil MCPワークフロー |

## 主な制約

| 項目 | 値 |
|------|-----|
| 対象ユーザー | FREEMIND社 全60名（4部署） |
| チーム | 13名（若手中心、PO/TL 1名、SM 1名、企画リード 1名、開発10名） |
| 予算 | インフラ+AI API 月額50 USD |
| 開発手法 | アジャイル/スクラム、2週間スプリント |
| 期間 | 約6ヶ月（硬い締切ではない） |
| 最優先事項 | メンバーの学び・成長 |
| コンピテンシー | 誠実、当事者意識、協働、挑戦への意欲、創造的思考 |
